# GCMP 聊天窗口文件修改跟踪功能

## 🎉 功能说明

我们已经为 GCMP 添加了完整的聊天窗口文件修改跟踪功能！现在当你使用 `gcmp_applyDiffV2` 工具修改文件时，所有的修改历史都会被自动跟踪并显示在聊天窗口中。

## 🛠️ 主要功能

### 1. **Chat Participant 集成**

- 🤖 **`@gcmp-assistant`** - 专门的聊天助手，用于显示文件修改历史
- 📋 **修改历史跟踪** - 自动记录所有通过工具进行的文件修改
- 🔍 **文件引用** - 在聊天窗口中直接引用被修改的文件
- ⏰ **时间戳记录** - 精确记录每次修改的时间

### 2. **聊天命令**

- **`@gcmp-assistant /history`** - 查看详细的文件修改历史
- **`@gcmp-assistant /status`** - 查看当前修改状态和统计信息
- **`@gcmp-assistant`** - 查看帮助信息和可用功能

## 📖 使用方法

### 步骤 1: 使用工具修改文件

```
使用 gcmp_applyDiffV2 工具修改 src/example.ts 文件，添加新的函数
```

### 步骤 2: 查看修改历史

在聊天窗口中输入：

```
@gcmp-assistant /history
```

你将看到：

- 📄 **修改的文件列表**
- ⏰ **修改时间**
- 🔧 **修改来源**（工具、手动等）
- ✏️ **编辑详情**（修改行数、位置等）
- 🔗 **文件引用链接**（点击直接打开文件）

### 步骤 3: 查看当前状态

```
@gcmp-assistant /status
```

显示：

- 📊 **总修改次数**
- 🕐 **最近1小时的修改数量**
- 📄 **最后修改的文件**
- 🛠️ **可用工具列表**

## 🔄 工作流程

1. **自动检测** - 当 `gcmp_applyDiffV2` 工具被调用时自动检测聊天上下文
2. **记录修改** - 详细记录文件URI、编辑位置、修改内容等信息
3. **聊天集成** - 将修改信息存储到 Chat Participant 中
4. **历史显示** - 通过 `@gcmp-assistant` 在聊天窗口中显示修改历史
5. **文件引用** - 提供直接的文件访问链接

## 🔧 管理功能

### 清理历史记录

使用命令面板 (Ctrl+Shift+P) 或聊天窗口中的按钮：

```
> GCMP: 清理文件修改历史
```

### 最大历史记录数

- 默认保存最近 **50** 次修改记录
- 自动清理旧记录以保持性能

## 🚀 技术实现

### Chat Participant 架构

```typescript
GCMPChatParticipant
├── 文件修改事件记录
├── 聊天命令处理
├── 历史查询和显示
└── 文件引用生成
```

### 集成流程

```
Language Model Tool (gcmp_applyDiffV2)
    ↓ 检测聊天上下文
    ↓ 执行文件修改
    ↓ 记录修改信息
    ↓
Chat Participant (@gcmp-assistant)
    ↓ 存储修改历史
    ↓ 响应查询命令
    ↓ 显示格式化历史
```
