# Ghost 代码补全功能使用指南

Ghost 是 GCMP 扩展提供的智能代码补全功能，使用智谱 GLM-4.5-air 订阅模型提供实时代码建议。Ghost 通过 VS Code 的 InlineCompletionItemProvider API 集成，提供类似 GitHub Copilot 的行内补全体验。

## 功能特点

- ✨ **行内补全**: 使用 InlineCompletionItemProvider，提供原生的行内代码建议
- 🤖 **AI 驱动**: 基于智谱 GLM-4.5-air 模型的智能代码生成
- ⚡ **流式响应**: 实时生成代码建议，快速响应
- 🎯 **上下文感知**: 分析前后文代码，生成最合适的补全
- 💰 **成本追踪**: 实时显示 API 调用成本

## 前置条件

1. 安装 GCMP 扩展
2. 设置智谱 AI API 密钥：
   - 运行命令: `GCMP: 设置 智谱AI API密钥`
   - 或使用命令面板 (Ctrl/Cmd+Shift+P): `gcmp.zhipu.setApiKey`
3. 确保使用编程套餐的 API 密钥（编程套餐提供更优惠的价格）

## 配置选项

在 VS Code 设置中搜索 `gcmp.ghost` 可以找到以下配置项：

### `gcmp.ghost.modelId`
- **类型**: string
- **默认**: "glm-4.5-air"
- **说明**: 使用的模型 ID。支持智谱编程套餐的所有模型。

### `gcmp.ghost.showStatusBar`
- **类型**: boolean
- **默认**: true
- **说明**: 在状态栏显示 Ghost 状态信息（模型、成本等）。

## 使用方法

Ghost 作为 VS Code 行内补全提供商运行，使用方式类似 GitHub Copilot。

### 自动触发（推荐）

1. 在编辑器中正常编写代码
2. 当你停止输入时，Ghost 会自动分析上下文
3. 如果有合适的建议，会以灰色文本形式显示在光标后
4. 按 `Tab` 键接受建议
5. 按 `Esc` 键拒绝建议
6. 继续输入会自动拒绝当前建议

**触发条件**：
- 当前行有至少 3 个字符
- 距离上次补全至少 2 秒
- 模型已就绪且有 API 密钥

**示例场景**：
```typescript
function calculateSum(numbers) {
    // 当你输入 "return " 后稍作停顿
    return |  // Ghost 会建议: numbers.reduce((a, b) => a + b, 0);
}
```

### 手动触发

1. 将光标放在需要补全的位置
2. 按 `Alt+\` （Windows/Linux）或 `Option+\` （macOS）
3. Ghost 会立即生成建议
4. 按 `Tab` 接受或 `Esc` 拒绝

### 部分接受

1. 当显示建议时，可以：
   - 按 `Ctrl+→` 只接受下一个单词
   - 按 `Ctrl+End` 接受到行尾
   - 继续输入部分接受匹配的内容

## 建议显示说明

Ghost 生成的补全建议会以行内灰色文本形式显示：

### 视觉效果
```typescript
function example() {
    return █  // 灰色建议文本会出现在这里
}
```

### 状态栏信息
状态栏右侧会显示 Ghost 状态：
- `$(sparkle) Ghost ✓` - 就绪状态
- `$(sparkle) Ghost ✗` - 未配置或错误
- `¥0.0001 (¥0.0001)` - 总成本（最后一次成本）

### 建议来源
- Ghost 使用上下文分析生成建议
- 建议长度取决于模型输出
- 通常为单行或多行代码块

## 工作原理

1. **触发检测**: 用户输入代码或手动触发（Alt+\）
2. **上下文收集**: Ghost 收集当前文件内容、光标位置、前后文代码
3. **提示构建**: 将上下文转换为 AI 可理解的提示词
4. **模型调用**: 调用智谱 GLM-4.5-air 模型生成代码
5. **流式响应**: 实时接收模型输出
6. **文本清理**: 移除可能的 markdown 标记
7. **建议显示**: 以行内形式显示在编辑器中
8. **用户操作**: 用户可以接受、拒绝或部分接受

## 成本说明

Ghost 使用智谱 GLM-4.5-air 编程套餐模型，定价为：
- 输入: 约 ¥0.0001/千tokens
- 输出: 约 ¥0.0001/千tokens

状态栏会实时显示：
- 当前会话总成本
- 最后一次补全成本
- 使用的模型名称

每次补全的典型成本约 ¥0.0001-0.001 元。

## 与其他补全的协作

Ghost 与 VS Code 内置的补全和其他扩展和谐共存：

- **优先级**: Ghost 作为行内补全，不影响其他补全
- **触发时机**: 自动触发时有 2 秒间隔，避免过于频繁
- **共存模式**: 可以同时启用 Ghost 和其他代码补全扩展
- **快捷键**: 使用标准的 Tab/Esc 操作，不冲突

## 故障排除

### 没有看到 Ghost 建议

**可能原因**：
1. API 密钥未设置或无效
2. 网络连接问题
3. 当前行太短（少于 3 个字符）
4. 距离上次补全不到 2 秒
5. 模型生成失败

**解决方法**：
1. 检查输出面板（选择 "GCMP" 通道）查看错误信息
2. 验证 API 密钥：运行 `gcmp.zhipu.setApiKey`
3. 确保当前行有足够内容
4. 等待 2 秒后再次尝试
5. 查看状态栏是否显示 ✓

### Ghost 建议不出现

**可能原因**：
1. Ghost 正在生成中
2. 当前位置无法生成有效建议
3. 模型认为不需要补全

**解决方法**：
1. 等待几秒钟让 Ghost 完成生成
2. 尝试在不同位置触发补全
3. 使用 Alt+\ 手动触发
4. 检查日志输出

### 建议质量不佳

**改进方法**：
1. 在有更多上下文的位置触发补全
2. 添加注释描述预期行为
3. 考虑切换模型（修改 `gcmp.ghost.modelId`）
4. 提供更清晰的代码结构

### 补全响应太慢

**优化方法**：
1. 检查网络连接
2. 使用更快的模型（如 glm-4-flash）
3. 减小文件大小（Ghost 读取前后约 60 行）
4. 检查是否有其他扩展影响性能

## 最佳实践

1. **适时触发**: 在编写完整的代码行后稍作停顿，让 Ghost 有时间分析
2. **提供上下文**: 编写清晰的函数签名、注释和变量名
3. **渐进式接受**: 使用 Ctrl+→ 逐词接受，而不是一次性接受全部
4. **及时拒绝**: 如果建议不合适，及时按 Esc 拒绝
5. **监控成本**: 注意状态栏的成本显示，避免过度使用

## 注意事项

1. **单次请求限制**: Ghost 一次只处理一个补全请求，并发请求会被忽略
2. **频率限制**: 自动触发有 2 秒间隔，避免过于频繁的 API 调用
3. **上下文大小**: Ghost 读取光标前 50 行和后 10 行代码作为上下文
4. **模型限制**: GLM-4.5-air 最大支持 128K 输入 tokens
5. **API 配额**: 注意智谱 AI 的 API 调用限制和配额
